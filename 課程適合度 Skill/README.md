# 課程適合度 Skill

一個給 Claude Code 用的 Agent Skill：丟一個線上課程網址給它，它會研究課程與講師、反問你幾個關於自身程度與目標的問題，最後輸出一個 **1~100 的適合度分數**與合適／不合適的具體理由。

核心定位是**適合度為主、犀利打假為輔**——不只判斷「這堂課好不好」，而是判斷「這堂課適不適合『你』」，同時把講師可信度、可複製性與割韭菜風險納入扣分項。

> 打假面向（講師頭銜還原、業績佐證、可複製性、匿名論壇負評）的思路參考自
> [cyesuta/garlic-chives](https://github.com/cyesuta/garlic-chives)，
> 並在此基礎上加上「與使用者程度／目標／預算的匹配度」，改寫成可執行的 Skill workflow。

## 觸發時機

當使用者提供課程／講座／訓練營網址並詢問類似問題時：

- 「這堂課適不適合我？」
- 「這門課值不值得買？」
- 「課程適合度」
- 「這堂課會不會割韭菜？」

## Workflow

| Step | 做什麼 |
|---|---|
| 0 | 取得課程網址（可一次多個：官網、報名頁、講師頁） |
| 1 | 研究課程本身——大綱、時數、形式、宣稱成果、價格、退費政策、講師姓名 |
| 2 | 研究講師背景——學經歷關聯度、頭銜還原、年資與業績佐證、匿名論壇負評、可複製性 |
| 3 | 用 `AskUserQuestion` 問使用者：目前程度、預期目標、可投入時間、預算敏感度 |
| 4 | 依五個維度評分加總成 1~100 分 |
| 5 | 在對話中輸出 Markdown 報告（**不寫檔、不進 vault**） |

研究方式優先使用 **Claude in Chrome**（`mcp__claude-in-chrome__*`），失敗 2~3 次後退回 `WebFetch` / `WebSearch`，不無限重試。

## 評分維度

| 維度 | 滿分 | 評什麼 |
|---|---|---|
| 講師可信度與關聯度 | 25 | 學經歷是否硬核相關、頭銜／業績是否經得起還原與查證 |
| 課程含金量與可複製性 | 25 | 硬技能還是空泛心法、是否只是免費資訊重新包裝、有無隱藏成本 |
| 與你「程度」的匹配 | 20 | 太淺浪費錢 / 太深跟不上 / 剛好 |
| 與你「目標」的匹配 | 20 | 課程成果能否真的帶你到你想去的地方 |
| 成本／時間投報比 | 10 | 以預算與每週可投入時間衡量是否划算、有無更便宜的替代方案 |

分數區間：

- **80–100** 高度適合，值得入手
- **60–79** 可考慮，但有需留意的條件
- **40–59** 保留，疑慮明顯，建議先看替代方案
- **1–39** 不建議，割韭菜風險或不匹配程度高

## 設計原則

- 一律以**繁體中文（台灣用語）**輸出，語氣理性、犀利、不和稀泥。
- 不把行銷頁的好處直接當成優點；優點必須是「對這位使用者」成立的具體理由。
- 查不到的資訊標「未提供／無法查證」，**不編造**講師經歷或學員成果。
- 忽略帶優惠碼的聯盟行銷推薦文與平台官方好評。
- 明細五項加總必須等於最終分數，分數要可被解釋。

## 安裝

複製整個 skill 目錄到 Claude Code 的 skills 目錄：

```bash
cp -R "課程適合度 Skill/course-suitability" ~/.claude/skills/
```

（本 repo 另有 `skills/install.sh` 可批次安裝 `skills/` 底下的 skill；此 skill 目前放在獨立目錄，未納入該腳本。）
