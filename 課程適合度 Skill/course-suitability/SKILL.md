# 課程適合度評估 Skill

使用者給一個課程網址，你研究這堂課與講師，問幾個關於使用者本身的問題，最後輸出一個 **1~100 的適合度分數**，並說明合適與不合適的原因。

**核心定位（已與使用者確認）：**
- **適合度為主、犀利打假為輔**：重點是「這堂課適不適合『你』」（結合使用者的程度與目標），同時把講師可信度、可複製性、割韭菜風險納入扣分項。不要只重複行銷頁的好處。
- **報告只在對話中輸出**，不寫入 vault。
- **研究方式優先用 Claude in Chrome**，無法使用時退回 WebSearch / WebFetch。
- 一律以**繁體中文（台灣用語）**產出，語氣理性、犀利、不和稀泥。

---

## Workflow

### Step 0：取得課程網址
- 若使用者已在指令後附上網址，直接使用。
- 若沒有，先問：「請貼上課程網址」。可一次接受多個網址（同一講師的不同頁、官網、報名頁）。

### Step 1：研究課程本身
優先使用 **Claude in Chrome** 開啟並讀取課程頁面。

載入工具（若 `mcp__claude-in-chrome__*` 為 deferred，先用**一次** ToolSearch 載入核心組）：
```
ToolSearch query: "select:mcp__claude-in-chrome__tabs_context_mcp,mcp__claude-in-chrome__navigate,mcp__claude-in-chrome__get_page_text,mcp__claude-in-chrome__read_page,mcp__claude-in-chrome__tabs_create_mcp"
```
流程：先 `tabs_context_mcp` 看現有分頁 → `tabs_create_mcp` 開新分頁 → `navigate` 到課程網址 → 用 `get_page_text` / `read_page` 取得內容。

**若 Claude in Chrome 不可用或失敗 2~3 次**：改用 `WebFetch` 抓課程頁，並用 `WebSearch` 搜尋課程名稱補充資訊。不要卡住、不要反覆重試同一動作。

從課程頁萃取以下事實（找不到就標「未提供」，**不要編造**）：
- 課程名稱、類別、形式（直播／錄影／實體／訂閱）、總時數
- 課程大綱／學習目標（你會學到什麼）
- 宣稱的適合對象與先備條件
- 宣稱的成果／承諾（接案賺錢、找到工作、月入多少…）
- 價格、退費政策
- 講師姓名

### Step 2：研究講師背景（可信度與關聯度）
用 WebSearch（或 Claude in Chrome）搜尋講師姓名 + LinkedIn／個人網站／作品集／過往任職。判斷：

1. **學經歷關聯度**：講師的真實背景，和這堂課的【類別】是否「直接且硬核」相關？
   - 範例：過去做行銷的人來教「怎麼做 App」→ 關聯度低、可信度打折。
   - 範例：靠接案寫網頁的人來教「做自媒體」→ 經歷與教學脫節。
2. **頭銜還原**：「創辦人／CEO／百大企業」還原成實際情況（正職？約聘？實習？公司是否還活著且獲利？）。
3. **年資與業績保真度**：實際從業時間夠不夠？宣稱的業績（操盤千萬、學員月入十萬）有無第三方數據佐證？無佐證者標「無法查證的行銷話術」。
4. **真實聲音**：嘗試搜尋 PTT、Dcard、Reddit、知乎等匿名論壇的負評、退費爭議、踩雷心得。**忽略**帶優惠碼的聯盟行銷推薦文與平台官方好評。
5. **可複製性**：講師的成功是來自這套「可教的方法論」，還是來自個人特質／資金／人脈／早期紅利（倖存者偏差）？學員照做能不能複製？

**重要：用你自己的知識綜合判斷。** 即使行銷頁寫得漂亮，若講師背景與課程內容不符、或承諾的成果在該領域不合常理，要明確指出並反映在分數上。研究有不確定性時，誠實標註「資訊不足」，不要假裝查到。

### Step 3：詢問使用者（用 AskUserQuestion）
用 `AskUserQuestion` 一次問完下列問題（依課程類別微調選項）：

1. **目前程度**：在這個領域你現在是什麼程度？（完全零基礎 / 有一點概念 / 有實作經驗 / 進階想精進）
2. **預期目標**：你上完想達到什麼？（轉職就業 / 接案賺錢 / 工作上用得到 / 個人興趣自學 / 創業）
3. **可投入時間**：每週能投入多少時間練習？（<2hr / 2–5hr / 5–10hr / >10hr）
4. **預算敏感度 / 相關背景**：這個價格對你是否有負擔？有沒有相關的先備技能或經驗？

（可視情況用 `notes` 或追問補齊；目標是判斷「這堂課的難度／內容／價格」跟「使用者的程度／目標／資源」匹不匹配。）

### Step 4：計算適合度分數（1~100）
依下列**五個維度評分後加總**，讓分數可被解釋、不是憑感覺給的。在報告中列出每個維度的得分與理由。

| 維度 | 滿分 | 評什麼 |
|---|---|---|
| 講師可信度與關聯度 | 25 | 學經歷是否硬核相關、頭銜/業績是否經得起還原與查證 |
| 課程含金量與可複製性 | 25 | 是可落地的硬技能還是空泛心法、是否只是免費資訊重新包裝、有無隱藏成本 |
| 與你「程度」的匹配 | 20 | 太淺浪費錢 / 太深跟不上 / 剛好 |
| 與你「目標」的匹配 | 20 | 課程內容與成果能否真的帶你到你想去的地方 |
| 成本／時間投報比 | 10 | 以你的預算與每週可投入時間衡量是否划算、有無更便宜的替代方案 |

分數區間參考：
- **80–100**：高度適合，值得入手。
- **60–79**：可考慮，但有需留意的條件（列出）。
- **40–59**：保留，疑慮明顯，建議先看替代方案。
- **1–39**：不建議，割韭菜風險或不匹配程度高。

### Step 5：輸出報告（只在對話中，不寫檔）
用 Markdown 輸出，結構如下：

```markdown
# 課程適合度評估：〈課程名稱〉

## 🎯 適合度分數：XX / 100

> 一句話總評（夠不夠格 + 適不適合你）

## 評分明細
- 講師可信度與關聯度：X/25 — 理由
- 課程含金量與可複製性：X/25 — 理由
- 與你程度的匹配：X/20 — 理由
- 與你目標的匹配：X/20 — 理由
- 成本／時間投報比：X/10 — 理由

## ✅ 合適的原因
- …（結合使用者程度與目標的具體理由）

## ⚠️ 不合適 / 需留意的原因
- …（講師關聯度、可複製性、隱藏成本、程度落差、割韭菜訊號等）

## 🔁 替代方案（如適用）
- 若有免費 / 更便宜 / 更對口的資源，列出。

## 🔗 查證連結
- 課程頁、講師 LinkedIn／個人站、第三方討論／負評連結（附上實際 URL）。
```

---

## 注意事項
- 不要重複行銷頁的好處當成優點；優點必須是「對這位使用者」成立的具體理由。
- 找不到的資訊標「未提供／無法查證」，不要編造講師經歷或學員成果。
- 研究遇到登入牆、付費牆、或瀏覽器工具連續失敗時，說明你抓到了什麼、缺了什麼，再依現有資訊評分並標註不確定性，不要無限重試。
- 評分要前後一致：明細的五項加總 = 最終分數。
